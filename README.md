# Млин — сайт з адмін-панеллю

Сайт ресторану/банкетного залу «Млин» (Дрогобич) на Node.js + Express + EJS.
Дані (меню, зала, контакти, новини, текст головної) зберігаються у файлі
`data/db.json` і редагуються через захищену паролем адмін-панель `/admin`.

## ⚠️ Важливо: чому не GitHub Pages

GitHub Pages віддає тільки готові статичні файли (HTML/CSS/JS) і **не вміє
запускати сервер**. Цей сайт — повноцінний Node.js-застосунок (динамічні
сторінки + адмінка + збереження змін), тому йому потрібен хостинг, що
запускає Node-процес. Підійде, наприклад:

- **Railway** (railway.app) — найпростіше, той самий підхід, що й для сайту Белуко
- **Render** (render.com) — теж є безкоштовний план
- Будь-який VPS з Node.js

Сам код можна (і варто) тримати на GitHub — просто деплой іде не через
GitHub Pages, а через підключення репозиторію до Railway/Render.

## Запуск локально

```bash
npm install
cp .env.example .env      # за бажанням зміни пароль адмінки
npm start
```

Сайт: http://localhost:3000
Адмінка: http://localhost:3000/admin/login (пароль за замовчуванням: `mlyn2026`)

## Деплой на Railway (рекомендовано)

1. Заливаєш цю папку як репозиторій на GitHub
2. На railway.app → New Project → Deploy from GitHub repo
3. У Variables додаєш `ADMIN_PASSWORD` (свій пароль) — `PORT` Railway підставить сам
4. Railway сам розпізнає Node.js і виконає `npm install` + `npm start`
5. Готово — отримуєш постійне посилання типу `https://mlyn-restoran.up.railway.app`

## Важливо про зберігання даних

Дані зараз пишуться у файл `data/db.json`. На Railway файлова система
**не постійна** при кожному новому деплої (як і було з NeDB у проєкті
Зони) — зміни через адмінку зберігатимуться, поки контейнер живий, але
зникнуть при наступному деплої коду. Для довготривалого продакшну краще
згодом перенести `db.json` на MongoDB Atlas (безкоштовний кластер) —
структура коду це дозволяє зробити без переписування сторінок, лише
модуль `db.js` потрібно замінити на звернення до Mongo.

## Структура

```
server.js          — усі маршрути (публічні + адмінка)
db.js               — читання/запис data/db.json
data/db.json        — увесь контент сайту
views/              — EJS-шаблони сторінок
views/admin/        — логін і дашборд адмінки
public/             — CSS, JS, зображення
```

## Маршрути

| Сторінка | URL |
|---|---|
| Головна | `/` |
| Меню | `/menu` |
| Зала | `/hall` |
| Новини | `/news`, окремий пост `/news/:slug` |
| Контакти | `/contacts` |
| Адмінка | `/admin` (вхід — `/admin/login`) |
